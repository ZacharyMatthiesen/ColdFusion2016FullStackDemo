Exercise 2

 

Objectives: To learn about cfinclude, SQL formatting and security measures and cflock using the Employee Contacts Application.

 

 

Requirements

 

1.       Consolidate code from multiple templates.

2.       Make SQL more transportable between database types.

3.       Protect the database from hacks.

4.       Make the SQL code run faster.

5.       Make SQL more readable.

6.       Lock all access to persistent scope variables.

 

Implementation

 

1.       Find code that is in multiple templates (cfm files) and replace with includes of external files.

 

Example:          add a header and footer to every page using <cfinclude>.

 

 

2.       Use of <cfqueryparam> makes SQL code more transportable between database types (Oracle & MySQL). 

 

3.       Use of <cfqueryparam> protects the database from hacks.

 

4.       Use of <cfqueryparam> makes SQL code run faster.

 

5.       Format SQL so it has 1 field per row for readability.

 

Example:          Select   name

                                    ,name

                                    ,name

                        From    name

                        Where   name = name

 

6.       Use <cflock> to lock access to persistent scope variables.

 

Tip:                  session.var = “string”     would be exclusive

                        xMyVar = session.var    would be read-only
